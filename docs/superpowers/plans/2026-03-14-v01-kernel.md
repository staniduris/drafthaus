# Drafthaus v0.1 — The Kernel

## Goal
`./drafthaus serve mysite.draft` → a fast, beautiful website from a single binary + single SQLite file. Nothing else.

## What v0.1 IS
- A Go binary (~20MB) with zero external dependencies
- A `.draft` file format (SQLite) containing: entity types, entities, blocks, relations, assets, views, design tokens
- A component-tree renderer that turns structured data into HTML + CSS
- An HTTP server with automatic routing from entity type definitions
- A CLI with `init`, `serve`, and `export` commands
- Built-in design token system (no CSS authoring)
- ~25 rendering primitives (Stack, Columns, Grid, Text, Image, etc.)

## What v0.1 IS NOT
- No AI layer
- No WASM plugins
- No CRDT collaboration
- No projection adapters (Google, social, email)
- No admin UI (API only — admin UI is v0.2)
- No user authentication
- No multi-tenancy

## How people create a .draft file in v0.1

```bash
# Option 1: CLI scaffolding
drafthaus init mysite --template cafe
# Creates mysite.draft with entity types, sample content, views, tokens

# Option 2: API
drafthaus serve mysite.draft
# Then use the REST API to create entity types, entities, views
curl -X POST localhost:3000/_api/types -d '{"name":"Product","fields":[...]}'

# Option 3: Import (stretch goal)
drafthaus import --from wordpress-export.xml mysite.draft
```

Templates ship embedded in the binary: `blank`, `cafe`, `portfolio`, `blog`, `business`.

### SQLite Schema

```sql
-- .draft file schema v1
PRAGMA journal_mode=WAL;
PRAGMA foreign_keys=ON;

CREATE TABLE dh_meta (
    key   TEXT PRIMARY KEY,
    value TEXT
);
-- seed: INSERT INTO dh_meta VALUES ('schema_version', '1');
-- seed: INSERT INTO dh_meta VALUES ('drafthaus_version', '0.1.0');

CREATE TABLE entity_types (
    id         TEXT PRIMARY KEY,
    name       TEXT NOT NULL,
    slug       TEXT NOT NULL UNIQUE,
    fields     TEXT NOT NULL,  -- JSON: [{name, type, required, default, validation}]
    icon       TEXT,
    routes     TEXT,           -- JSON: {list: "/blog", detail: "/blog/{slug}"}
    created_at INTEGER NOT NULL DEFAULT (unixepoch()),
    updated_at INTEGER NOT NULL DEFAULT (unixepoch())
);

CREATE TABLE entities (
    id         TEXT PRIMARY KEY,
    type_id    TEXT NOT NULL REFERENCES entity_types(id) ON DELETE CASCADE,
    data       TEXT NOT NULL DEFAULT '{}',  -- JSON: field values
    slug       TEXT,
    status     TEXT NOT NULL DEFAULT 'draft' CHECK (status IN ('draft','published','archived')),
    position   REAL NOT NULL DEFAULT 0,
    created_at INTEGER NOT NULL DEFAULT (unixepoch()),
    updated_at INTEGER NOT NULL DEFAULT (unixepoch())
);
CREATE UNIQUE INDEX idx_entities_type_slug ON entities(type_id, slug);
CREATE INDEX idx_entities_type_status ON entities(type_id, status);

CREATE TABLE blocks (
    id        TEXT PRIMARY KEY,
    entity_id TEXT NOT NULL REFERENCES entities(id) ON DELETE CASCADE,
    field     TEXT NOT NULL,
    type      TEXT NOT NULL,
    data      TEXT NOT NULL DEFAULT '{}',  -- JSON
    position  REAL NOT NULL DEFAULT 0,
    parent_id TEXT REFERENCES blocks(id) ON DELETE CASCADE
);
CREATE INDEX idx_blocks_entity_field ON blocks(entity_id, field);

CREATE TABLE relations (
    source_id     TEXT NOT NULL REFERENCES entities(id) ON DELETE CASCADE,
    target_id     TEXT NOT NULL REFERENCES entities(id) ON DELETE CASCADE,
    relation_type TEXT NOT NULL,
    position      REAL NOT NULL DEFAULT 0,
    metadata      TEXT DEFAULT '{}',
    PRIMARY KEY (source_id, target_id, relation_type)
);
CREATE INDEX idx_relations_target ON relations(target_id, relation_type);

CREATE TABLE assets (
    id       TEXT PRIMARY KEY,
    hash     TEXT NOT NULL UNIQUE,
    name     TEXT NOT NULL,
    mime     TEXT NOT NULL,
    size     INTEGER NOT NULL,
    data     BLOB NOT NULL,
    metadata TEXT DEFAULT '{}',  -- JSON: {width, height, alt, blurhash}
    created_at INTEGER NOT NULL DEFAULT (unixepoch())
);

CREATE TABLE views (
    id         TEXT PRIMARY KEY,
    name       TEXT NOT NULL UNIQUE,  -- e.g. "BlogPost.detail", "Product.list"
    tree       TEXT NOT NULL,         -- JSON: component tree
    version    INTEGER NOT NULL DEFAULT 1,
    created_at INTEGER NOT NULL DEFAULT (unixepoch()),
    updated_at INTEGER NOT NULL DEFAULT (unixepoch())
);

CREATE TABLE tokens (
    id    TEXT PRIMARY KEY DEFAULT 'default',
    data  TEXT NOT NULL,  -- JSON: full design token set
    updated_at INTEGER NOT NULL DEFAULT (unixepoch())
);

CREATE TABLE versions (
    entity_id  TEXT NOT NULL REFERENCES entities(id) ON DELETE CASCADE,
    version    INTEGER NOT NULL,
    data       TEXT NOT NULL,   -- JSON: full entity snapshot
    blocks     TEXT,            -- JSON: blocks snapshot
    changed_at INTEGER NOT NULL DEFAULT (unixepoch()),
    PRIMARY KEY (entity_id, version)
);
```

```go
// Example: Stack primitive
func stackPrimitive(n *Node, ctx *RenderContext) (string, error) {
    gap := resolveTokenValue(n.Props["gap"], ctx.Tokens)
    var buf strings.Builder
    buf.WriteString(`<div class="dh-stack"`)
    if gap != "" {
        buf.WriteString(fmt.Sprintf(` style="gap:%s"`, gap))
    }
    buf.WriteString(`>`)
    for _, child := range n.Children {
        html, err := ctx.Render(child)
        if err != nil {
            return "", err
        }
        buf.WriteString(html)
    }
    buf.WriteString(`</div>`)
    return buf.String(), nil
}
```

### Data Binding

Resolves `bind` expressions against entity data:

```go
// internal/render/bind.go

// BindContext holds the resolved entity data available for binding
type BindContext struct {
    Entity    *graph.ResolvedEntity
    Entities  []*graph.ResolvedEntity // for list views
    Tokens    *draft.Tokens
}

// ResolveBind resolves a bind expression like "name" or "authored_by.name"
// against the entity data
func ResolveBind(expr string, ctx *BindContext) (any, error)

// BindTree walks the component tree and resolves all bind expressions,
// producing a tree with concrete values in Props
func BindTree(root *Node, ctx *BindContext) (*Node, error)
```

Bind expressions:
- `"name"` → `entity.Data["name"]`
- `"authored_by.name"` → first entity in `relations["authored_by"]`, then `.Data["name"]`
- `"has_image.first"` → first relation of type `has_image`, resolved to asset URL
- `"body"` on a RichText node → loads blocks for field "body"

### Full Pipeline

```go

    registry *Registry
    resolver *graph.Resolver

}

func NewPipeline(store draft.Store) *Pipeline
```

RenderPage flow:
1. Load tokens from store
2. Parse view tree JSON into Node tree
3. Bind tree against entity data
4. Walk tree, render each node via registry
5. Generate CSS from tokens
6. Generate meta tags
7. Wrap in HTML document shell (`<!DOCTYPE html>`, `<head>`, `<body>`)
8. Return complete HTML bytes

```go
func (r *Router) Match(path string) (*Match, bool)
```

### CLI Commands

```
drafthaus init <name> [--template <template>]
    Creates a new .draft file with the given name.
    Templates: blank, blog, cafe, portfolio, business
    Default: blank

drafthaus serve <file.draft> [--port 3000] [--host 0.0.0.0]
    Starts the HTTP server serving the .draft file.
    Hot-reloads on .draft file changes (via fsnotify on the SQLite file).

drafthaus export <file.draft> <output-dir>
    Exports the site as static HTML files.
    Walks all entity routes, renders each, writes to output-dir.
    Copies assets. Generates sitemap.xml and robots.txt.
```
